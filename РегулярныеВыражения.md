## Кирилица в регулярках

Некоторые (php, вроде python) понимают что такое буквы (\w) нормально. Но JS и VSC до кучи считает не ASCII символы небуквами. Добрый ИИ предлагает такое решение:
>**Символьные классы Unicode**: VS Code поддерживает стандарт Unicode. Для поиска любых кириллических символов используйте: `\p{L}` (любая буква) или более специфично: `\p{Script=Cyrillic}`

Для поиска слова без цифр соответственно `\b\w+\b` превращается в `(?<!\p{L})\p{L}+(?!\p{L})` с цифрами (типа наименования переменной) `(?<!\p{L})(\w|\p{L})+(?!\p{L})`
Не аски цифры тоже наверное можно через диапазоны юникода, но надеюсь 1С до такого не дойдет.
## Просмотр вперед и назад

Еще может возникнуть необходимость найти какое-то место в тексте, но не включая найденное слово в выборку. Для этого мы «просматриваем» окружающий текст.

|                   |                                           |                      |                                                               |
| ----------------- | ----------------------------------------- | -------------------- | ------------------------------------------------------------- |
| **Представление** | **Вид просмотра**                         | **Пример**           | **Соответствие**                                              |
| (?=_шаблон_)      | Позитивный просмотр вперёд                | Блюдо(?=11)          | ~~Блюдо1~~<br><br>Блюдо11<br><br>Блюдо113<br><br>~~Блюдо511~~ |
| (?!_шаблон_)      | Негативный просмотр вперёд (с отрицанием) | Блюдо(?!11)          | Блюдо1<br><br>~~Блюдо11~~<br><br>~~Блюдо113~~<br><br>Блюдо511 |
| (?<=_шаблон_)     | Позитивный просмотр назад                 | (?<=Ольга )Назина    | Ольга Назина<br><br>~~Анна Назина~~                           |
| (?<!шаблон)       | Негативный просмотр назад (с отрицанием)  | (см ниже на рисунке) | ~~Ольга Назина~~<br><br>Анна Назина                           |
|                   |                                           |                      |                                                               |
### Пример
`Справочник(?:и)?\.Настройки(?!.*// ок$).*$` - описание:
- `Справочник(?:и)?` — "Справочник" или "Справочники"
- `\.Настройки` — буквально ".Настройки"
- `(?!.*// ок$)` — **негативный просмотр вперёд**: убеждаемся, что **всюду до конца строки** нет `// ок` в конце
- `.*$` — всё остальное до конца строки
